Controls (quick reference)

  • Grip (side squeeze) — grab / release a cartridge, the gamepad, or (in edit mode) a prop. Never sent to the game.
  • Thumbstick — walk, when you're not holding the gamepad.
  • Trigger on the wall Menu panel (left wall) — click a menu button. Drop the gamepad first, point, and pull.
  • Hold the gamepad in one hand to play: trigger / face buttons / thumbstick = game input, per the wall Controls card.

Part A — Single headset

  1. Enter VR. Open the single-headset URL, tap Enter VR. The 3D room renders — shelves, a console, a CRT TV, wall panels.
  2. Boot a game. Grip a cartridge off a shelf and drop it onto the console. It boots on the TV. If pixels appear, COOP/COEP + SharedArrayBuffer + the WASM core all work.
  3. Play it — controller gamepad. Grip the gamepad, hold it, and play: thumbstick = D-pad, trigger/face buttons act per the Controls card. Confirms the Quest exposes one live inputSource.gamepad per held controller.
  4. Menu raycast — modes. Drop the gamepad. Point at the wall Menu panel and pull the trigger to click. Toggle Show Controls and Show Debug.
  5. Move mode. Menu → Move. Grip a shelf or poster, drag it somewhere new, release. It stays put.
  6. Change mode. Menu → Change. Grip-select a poster, then Cycle Selected → its art changes. Grip-select a shelf, Cycle Selected → its game collection swaps. Try Wallpaper / Floor / Lighting — the room repaints live.
  7. Add mode. Menu → Add. Add a Shelf, Console, Poster, and Portal — each spawns in front of you and is immediately grabbable.
  8. Portal + Export. Walk into the portal you added → it navigates to another room. Back in the menu, Export Room downloads / copies the room JSON.

Part B — Two headsets M0 M1

Both headsets open the same ?session=ROOM URL (with different nick). For the basic test put both on the same Wi-Fi (STUN covers that; TURN is only for the cross-network case below).

  1. Presence. M0 Each of you sees the other as an avatar — floating head + two hands + a nameplate. Move your head and hands; the other sees it track.
  2. In-VR voice + mic grant. NEW On each headset: Menu → Voice: Off. Approve the Quest microphone prompt. The label flips to Voice: On. Talk — you hear each other, and the voice is spatial (quieter as you walk apart, panned by direction). Click Voice again → Voice: Muted. The key new check: does the Quest browser grant the mic during an XR session?
  3. Shared TV. M0 One player boots a game (grip cart → console). Both TVs converge on that game.
  4. Held-object ghost. M0 One player grips a cartridge and holds it. The other sees a translucent ghost cartridge in that player's hand.
  5. Host video stream. M1.2 After the host boots, the other player's TV shows the host's actual gameplay frames (a live video of the host's screen), not a separate copy.
  6. Host-authoritative input. M1.1 The non-host holds their gamepad and presses buttons → it drives the host's game. Both TVs show the host's game reacting to the remote player.
  7. Watcher core pauses / resumes. NEW While watching, the non-host runs no game of its own (its TV just mirrors the host). To confirm the resume path: the non-host boots their own game (grip a different cart → console) → they take over as host, their core resumes, and the first player now watches them.

Optional / advanced

  1. Cross-network (TURN). OPTIONAL Put the two headsets on different networks (e.g. one on a phone hotspot). Without TURN, voice/video may fail to connect (symmetric NAT). Re-open both with the &turn=…&turnUser=…&turnCred=… params → they should connect via the relay. Requires the coturn server to be set up first (see deploy/coturn.conf.example).
  2. Local ROM folder. OPTIONAL Does the “ROM folder…” button appear and work in the Quest browser (File System Access API)? Known unknown — note what happens.

What to report back
