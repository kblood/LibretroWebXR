# LWX Frontline Fury

A CC0 **"Operation Wolf"-style** on-rails two-gun light-gun shooter for the SNES,
built for LibretroWebXR. **Original content only** ‚Äî this is the *genre* (on-rails,
advance through waves, two guns, shoot off-screen to reload), authored fresh; no
copyrighted Operation Wolf assets, levels or art are used.

You advance through **stages** of an on-rails firefight: enemy soldiers march in
from the top and advance toward the **front line** at the bottom. Aim your gun and
pull the trigger to drop them before they arrive. Clear a stage's quota and you
**advance** to the next, tougher stage (faster enemies, bigger quota). Enemies that
reach the front line cost you **health**; run out and it's **GAME OVER**. Two-player
co-op: each player has an independent crosshair and score over a shared health bar.

**Off-screen shot = RELOAD** (the iconic Operation Wolf mechanic): each gun has an
8-round magazine. Firing on-screen spends a round; firing **off-screen** refills the
magazine. An empty magazine can't shoot ‚Äî reload at the right moment.

## Two guns on the SNES: the Konami Justifier

The SNES light gun that supports **two simultaneous guns** is the **Konami Justifier**
(Lethal Enforcers). snes9x device ids: **516** (Justifier, gun 1) / **772**
(Justifier2, gun 2), both daisy-chained on **controller port 2**. Like the Super
Scope it is **position-latched, not beam-timed**: when a gun's SELECT line is active,
snes9x latches that gun's aimed pixel into the PPU H/V counters (`OPHCT`/`OPVCT`
`$213C`/`$213D` ‚Äî the same gun-latch the Super Scope uses), so the ROM reads a
**stable coordinate** with no whole-frame photodiode polling.

**Justifier read protocol** (verified against the snes9x `controls.cpp` we ship):
each `$4016` strobe **toggles** which gun (0/1) latches its position to the PPU; after
the strobe, 32 serial bit-reads of `$4017` return a 24-bit signature (`0xaa7000`,
present iff a Justifier is connected) followed by 8 button bits for both guns
(TRIGGER/START/SELECT). So the ROM ping-pongs: one frame select+read gun 0, the next
gun 1 ‚Äî exactly how Lethal Enforcers does it.

PVSnesLib has **no** Justifier driver (only Super Scope), so the Justifier path here
is hand-written raw 65816-via-C register I/O (`jf_*` in `opwolf.c`). To keep the game
both Operation-Wolf-correct **and verifiable today**, the input layer is abstracted
behind `read_gun(p)` with two backends, chosen at boot by which device the frontend
attached:

| Backend | Device | Status |
|---|---|---|
| **Super Scope** (`GUN_SCOPE`) | 260, port 2 | The **verifiable** single-gun path today ‚Äî PVSnesLib `detectSuperScope()` / `scope_*`. The shipped single-mouse `rwebinput` feeds this. Runs the full game loop with a real latched coordinate. |
| **Justifier** (`GUN_JUST`) | 516 (port 1) / 772 (port 2) | Two-gun co-op via the raw reader above. The per-port multiport `rwebinput` patch (`docs/patches/rwebinput-lightgun-multiport.diff`) has **landed and is verified** ‚Äî each gun gets its own pointer and scores independently (see Verifying). Both guns are read 30 Hz each (multiplexed by SELECT) ‚Äî the hardware-faithful maximum; see below. |

The game **auto-detects**: it tries `detectSuperScope()` first; if a Super Scope
answered it runs single-gun, otherwise it strobes for the Justifier `0xaa7000`
signature and runs two-gun co-op.

## Build

```
node scripts/make-snes-opwolf.mjs      # from the repo root
```

Same **PVSnesLib template-fill** toolchain and lookup as `games/snes-scope` /
`games/snes-demo`. The build script **generates `sprites.bmp`** (soldier + two
crosshairs + hit-burst) procedurally before running `make`, converts it with
`gfx4snes`, compiles `opwolf.c` through `816-tcc ‚Üí 816-opt ‚Üí constify ‚Üí wla-65816 ‚Üí
wlalink`, copies the ROM to `public/roms/freeware/lwx-snes-opwolf.sfc`, and removes
the build intermediates. LoROM, SlowROM, ROM-only, 256 KB. Internal title
`LWX FRONTLINE FURY`.

## Controls

- **Aim** the gun at the screen and **pull the trigger** to shoot an enemy.
- **Shoot off-screen** to **reload** your magazine.
- At the title / game-over screen, **shoot the centre** to calibrate (Super Scope) +
  start.
- In VR: grab a light-gun prop ‚Äî the frontend boots snes9x with the gun on port 2
  (`"lightgun": true` in the manifest) and feeds your aim via
  `EmulatorClient.sendLightgun()`. Two guns = `"twoGun": true` + the Justifier config
  (`SYSTEMS.snes.lightgun2`) on the multiport core (live + verified).

## Verifying

`tmp/verify-opwolf-snes9x.mjs` drives the **real shipped snes9x core** headlessly
through `sendLightgun()` (the VR gun prop's entry point), on **both** boot paths
(`BOOT=pick` and `BOOT=shelf`): calibrate on centre ‚Üí enemies advance, an on-target
shot **kills an enemy (green flash)**, and an **off-screen shot reloads** so a kill
works again afterward. **5/5** on both paths.

This exercises the full game loop and the per-gun **logic** through the single-gun
Super Scope backend (the path a single-mouse `rwebinput` feeds).

**True two-gun co-op is now verified** by `tmp/verify-twogun-opwolf-snes9x.mjs`
against the **patched** snes9x core: it drives gun A (libretro port 1, device 516)
and gun B (port 2, device 772) to two distinct points via the per-port multiport
setter (`rwebinput_set_lightgun(port,‚Ä¶)`) and proves the core's blue/magenta
crosshairs each follow their **own** port (independent aim + per-port isolation),
**and** that each gun scores an in-game kill through this ROM's raw Justifier
reader. **9/9.**

### Hardware limit (why not "both guns per frame")

snes9x (faithful to real Lethal Enforcers) has exactly **one** pair of PPU gun-latch
counters and **one** serial button byte, both multiplexed by the `JUSTIFIER_SELECT`
line that toggles on every `$4016` strobe. So **both** a gun's position **and** its
trigger are inherently **30 Hz-per-gun** ‚Äî one frame reads gun 0, the next reads gun
1. Reading both guns' *fresh* state in a *single* frame is physically impossible on
this hardware; the ping-pong reader is the faithful maximum. Each gun's last reading
is held between updates, so co-op scoring uses both guns continuously (‚â§1 frame
stale ‚Äî imperceptible for a rail shooter).

## Notes / follow-ups

- **Port topology ‚Äî resolved.** `src/systems.js`'s SNES `lightgun2` config seats the
  two guns on **separate** libretro ports (516 on port 1, 772 on port 2). That is
  **correct**: snes9x reads the two Justifier guns from two distinct libretro ports
  at the `input_state_cb` boundary; the one-port daisy-chain / SELECT strobe is
  internal to snes9x's emulation, not at the frontend boundary. Verified by the
  two-gun harness above.
- Crosshair tile palette: P1 should be cyan / P2 pink; under the single-Super-Scope
  path the crosshair currently reads magenta (a sprite-palette index nuance). Purely
  cosmetic ‚Äî hit/reload/advance logic is unaffected (verified 5/5).
