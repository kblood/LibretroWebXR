# LWX Beam Dodger (Atari 2600)

A tiny CC0 Atari 2600 game for LibretroWebXR test content. One screen: you pilot
a ship (`player0`) along the bottom rail; a beam (`player1`) falls from the top
in a random column. Move left/right to dodge it. Each beam you survive adds to
the `score`; the beam speeds up a little as your score climbs (capped so it
stays dodgeable). One hit flashes the screen red and resets the run.

This is the **batari Basic (bB)** workflow from
`docs/research/atari2600-game-creation.md`: the brutal race-the-beam TIA timing
is delegated to bB's standard kernel, and the only file we author is `game.bas`
(game logic using bB's documented statements: `player0`/`player1` sprites,
`playfield`, `joy0left`/`joy0right`, `score`, `collision(...)`, `drawscreen`).
That split is what makes an AI-authored 2600 game reliable.

## Controls

| Input (RetroPad / joystick) | Action |
|---|---|
| Left / Right | Move the ship |

## Build

```
node scripts/make-atari-dodger.mjs      # from the repo root
```

Needs **batari Basic 1.8** installed at `C:\Atari2600\bB` (set `BB_HOME` to
override). The script:

1. Ensures a *working* `dasm.exe` is in the bB dir — the dasm bundled with bB 1.8
   (a 2.20.15-SNAPSHOT) **segfaults on Windows** and silently emits a 0-byte
   ROM, so the script swaps in the stable **dasm 2.20.14.1** win-x64 release
   (downloading it once to `C:\Atari2600\dasm-stable\`).
2. Copies `game.bas` into a scratch temp dir, sets the `bB` env var + PATH, and
   runs `2600bas.bat game.bas` (bB writes its outputs next to the source).
3. Verifies the produced `game.bas.bin` is a non-empty 2K/4K cart and copies it
   to `public/roms/freeware/lwx-atari-dodger.a26`.

Output: a standard **4K** (4096-byte) ROM for the `stella2014` core.

## Install batari Basic (one-time)

Download `bB-1.8-win-x64.zip` from
<https://github.com/batari-Basic/batari-Basic/releases/tag/v1.8> and extract so
that `C:\Atari2600\bB\2600bas.bat` exists. **Zip-extract only** — do not run the
interactive `install_win.bat`; the build sets the `bB` / PATH env vars itself.
(bB v1.9 ships a WASM build only; the v1.8 zip is the last native Windows
release.)
