# LWX Scope Range

A CC0 SNES **Super Scope** light-gun shooting gallery for LibretroWebXR — the
position-based companion to the NES Zapper game (`games/nes-gallery`). A bullseye
target pops up at random spots on a dark field; aim the Super Scope and pull the
trigger. On target = a HIT (+1 and a faster next target, green flash); fire on the
dark field or let a target time out = a MISS (red flash); three misses ends the
game. Standard **LoROM** `.sfc` (256 KB) that the libretro **snes9x** core runs
with the Super Scope on controller port 2.

## Why the Super Scope (not the NES Zapper) for a position game

The NES Zapper is a photodiode — its "light" bit is only true for the few cycles
the CRT beam crosses the muzzle's pixel, so a Zapper game must spin-poll `$4017`
across the whole visible frame (see `games/nes-gallery`). The SNES Super Scope is
different: when the trigger is held, snes9x latches the PPU H/V beam counters to
the aimed pixel, so the ROM reads a **stable coordinate** (`OPHCT`/`OPVCT` =
screenX + 40, screenY + 1) on any frame — no beam-timing tricks. PVSnesLib wraps
all of it: `detectSuperScope()`, the `scope_*` state, and `scope_shoth /
scope_shotv` (the aimed position). A one-shot "shoot the centre dot" calibration
sets `scope_centerh/v` so `scope_shoth/v` come out in screen pixels (this also
cancels the core's fixed +40 H offset) — exactly what a real Super Scope game does.

## Build

```
node scripts/make-snes-scope.mjs      # from the repo root
```

Same **PVSnesLib template-fill** toolchain and lookup as `games/snes-demo` (see
that README for the PVSnesLib / GNU make / Git-for-Windows details and *Gotchas*).
The build script additionally **generates `sprites.bmp`** (the bullseye + aim-dot
sprite sheet) procedurally before running `make`, then converts it with `gfx4snes`,
compiles `scopegame.c` through `816-tcc → 816-opt → constify → wla-65816 →
wlalink`, copies the ROM to `public/roms/freeware/lwx-snes-scope.sfc`, and removes
the build intermediates.

## Controls

- **Aim** the Super Scope at the screen and **pull the trigger** to shoot.
- At the title / game-over screen, shoot the **centre dot** to calibrate + start.
- In VR: grab the light-gun prop — the frontend boots snes9x with the Super Scope
  on port 2 (`"lightgun": true` in the manifest) and feeds your aim as the gun
  position via `EmulatorClient.sendLightgun()`.

## Verifying

`tmp/verify-scope-snes9x.mjs` drives the **real shipped snes9x core** headlessly
through `sendLightgun()` (the VR gun prop's entry point): it calibrates on centre,
then asserts an on-target shot flashes the backdrop **green** (HIT) and a dark-field
shot flashes it **red** (MISS) — 5/5 on both the direct (`BOOT=pick`) and the real
manifest (`BOOT=shelf`) boot paths. The green/red hit/miss flash is both game-feel
and the verification signal.

## Notes

- LoROM, SlowROM, ROM-only, 256 KB (8 × 32 KB banks). Internal SNES header title
  `LWX SCOPE RANGE`, mapmode `0x20` (LoROM).
- Super Scope device id **260** on controller port index **1** (player 2), with
  `snes9x_superscope_crosshair="enabled"` — see `docs/LIGHTGUN_SUPPORT.md` and
  `src/systems.js` (`SYSTEMS.snes.lightgun`).
